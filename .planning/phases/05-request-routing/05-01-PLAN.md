# Phase 05 Plan 01: Request Routing — Engine and Serving Layer

## Objective

Wire up Chatterbox request routing across three layers:
1. **Engine rerouting** — two new `reroute_request_chatterbox*()` methods + extensions to `check_reroute_request_to_model()` in [`harmonyspeech_engine.py`](harmonyspeech/engine/harmonyspeech_engine.py)

3. **Serving layer** — register Chatterbox model types/groups in [`serving_text_to_speech.py`](harmonyspeech/endpoints/openai/serving_text_to_speech.py), [`serving_voice_embed.py`](harmonyspeech/endpoints/openai/serving_voice_embed.py), and [`serving_voice_conversion.py`](harmonyspeech/endpoints/openai/serving_voice_conversion.py)

- Serving layer `_TTS_MODEL_TYPES`, `_EMBEDDING_MODEL_TYPES`, `_VOICE_CONVERSION_MODEL_TYPES` do not yet include Chatterbox variants.

### Task 1 — Add reroute_request_chatterbox() to HarmonySpeechEngine

**File:** `harmonyspeech/engine/harmonyspeech_engine.py`

Add the following method to the `HarmonySpeechEngine` class, **after** `reroute_request_voicefixer()` (currently ending at line ~299) and **before** `check_reroute_request_to_model()` (line 300):

```python
def reroute_request_chatterbox(self, request: RequestInput):
    # Map sentinel name → TTS model_type
    _CHATTERBOX_TTS_TYPE_MAP = {
        "chatterbox": "ChatterboxTTS",
        "chatterbox_turbo": "ChatterboxTurboTTS",
        "chatterbox_multilingual": "ChatterboxMultilingualTTS",
    }

    if (isinstance(request, SpeechEmbeddingRequestInput) or
        (
            isinstance(request, TextToSpeechRequestInput) and
            request.input_audio is not None and
            request.input_embedding is None
        )
    ):
        # Route to embedding model (Step 1 of multi-step, or standalone embed)
        for cfg in self.model_configs:
            if cfg.model_type == "ChatterboxEmbedding":
                request.model = cfg.name
                break
    elif (
        isinstance(request, TextToSpeechRequestInput) and
        (request.input_audio is None)  # direct path or Step 2 after embedding
    ):
        # Route to the appropriate TTS variant
        tts_type = _CHATTERBOX_TTS_TYPE_MAP.get(request.requested_model)
        if tts_type:
            for cfg in self.model_configs:
                if cfg.model_type == tts_type:
                    request.model = cfg.name
                    break
```

**Checklist:**
- [ ] Method added to `HarmonySpeechEngine` class after `reroute_request_voicefixer()`
- [ ] `_CHATTERBOX_TTS_TYPE_MAP` dict maps all 3 TTS sentinels to their model_type strings
- [ ] `SpeechEmbeddingRequestInput` branch routes to `ChatterboxEmbedding`
- [ ] `TextToSpeechRequestInput` with `input_audio is not None` and `input_embedding is None` routes to `ChatterboxEmbedding`
- [ ] `TextToSpeechRequestInput` with `input_audio is None` routes to the correct TTS variant via map lookup

### Task 4 — Add Chatterbox block to check_forward_processing()

**File:** `harmonyspeech/engine/harmonyspeech_engine.py`

In `check_forward_processing()`, add the following new block **after** the existing `# Multi-Step TTS Processing (Harmonyspeech, OpenVoice)` block (which ends before the `# Multi-Step Audio Conversion Processing (VoiceFixer)` block):

```python
        # Multi-step TTS Processing (Chatterbox — embed then synthesize)
        if isinstance(input_data, TextToSpeechRequestInput) and requested_model in [
            "chatterbox",
            "chatterbox_turbo",
            "chatterbox_multilingual"
        ]:
            forwarding_request = input_data

            if isinstance(result.result_data, SpeechEmbeddingRequestOutput):
                # Embedding step finished — forward to TTS synthesize step
                # Mark as forwarded in scheduler
                new_status = RequestStatus.FINISHED_FORWARDED
                self.scheduler.update_request_status(result.request_id, new_status)
                # Embedding result becomes input_embedding; clear input_audio
                forwarding_request.input_audio = None
                forwarding_request.input_embedding = result.result_data.output
                self.add_request(result.request_id, forwarding_request)
```

**Key design notes:**
- This block handles the `input_audio → ChatterboxEmbedding → input_embedding → ChatterboxTTS*` chain
- No `SpeechTranscriptionRequestOutput` branch needed — Chatterbox does NOT use VAD/transcription
- No `SpeechSynthesisRequestOutput` branch needed — Chatterbox TTS is self-contained (no separate vocoder)
- The embed step returns `SpeechEmbeddingRequestOutput`; once received, the request is re-submitted with `input_audio=None` and `input_embedding` set to the embedding output
- On the second pass through `reroute_request_chatterbox()`, `input_audio is None` triggers the direct TTS route

**Checklist:**
- [ ] New Chatterbox block inserted after the HarmonySpeech/OpenVoice TTS block, before VoiceFixer block
- [ ] Block checks `TextToSpeechRequestInput` + `requested_model in ["chatterbox", "chatterbox_turbo", "chatterbox_multilingual"]`
- [ ] Only handles `SpeechEmbeddingRequestOutput` (no transcription or synthesis intermediate branches)
- [ ] Sets `new_status = RequestStatus.FINISHED_FORWARDED`
- [ ] Calls `self.scheduler.update_request_status(result.request_id, new_status)`
- [ ] Sets `forwarding_request.input_audio = None`
- [ ] Sets `forwarding_request.input_embedding = result.result_data.output`
