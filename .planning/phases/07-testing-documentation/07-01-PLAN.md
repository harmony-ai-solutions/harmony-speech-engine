## Task 1 — Fix pytest markers and add `cuda` marker in pyproject.toml

**File:** `pyproject.toml`

**Root-cause fix:** `pyproject.toml` currently has the `markers` list under `[tool.coverage.run]` (lines 61–66). However, pytest reads marker registrations **only** from `[tool.pytest.ini_options].markers`. With `--strict-markers` in `addopts`, any unregistered `@pytest.mark.*` marker causes test collection failure. The existing `unit`, `integration`, `e2e`, `slow` markers are also in the wrong section — this task fixes all of them at once by moving the full markers list into `[tool.pytest.ini_options]` and removing the misplaced copy from `[tool.coverage.run]`.

**Current `[tool.pytest.ini_options]` section (lines 46–55):**
```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
asyncio_mode = "auto"
addopts = [
    "--strict-markers",
    "-ra",
    "--cov=harmonyspeech",
    "--cov-report=term-missing",
    "--cov-report=xml",
]
```

**Current `[tool.coverage.run]` section (lines 57–66) — markers are here (WRONG location):**
```toml
[tool.coverage.run]
omit = [
    "harmonyspeech/modeling/models/*",
]
markers = [
    "unit: marks tests as unit tests (fast, fully mocked)",
    "integration: marks tests as integration tests (component interaction, partially mocked)",
    "e2e: marks tests as end-to-end tests (real models, slow, CPU-only)",
    "slow: marks tests that take > 30 seconds to run",
]
```

**Target `[tool.pytest.ini_options]` section:**
```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
asyncio_mode = "auto"
addopts = [
    "--strict-markers",

markers = [
    "unit: marks tests as unit tests (fast, fully mocked)",

    "e2e: marks tests as end-to-end tests (real models, slow, CPU-only)",
    "slow: marks tests that take > 30 seconds to run",
    "cuda: marks tests that require CUDA/GPU hardware",
]
```

**Target `[tool.coverage.run]` section (markers removed):**
```toml

    "harmonyspeech/modeling/models/*",
]
```

        requested_model="chatterbox",
        input_audio="base64audio==",
        input_embedding=None,

    engine.scheduler.get_request_input_data = MagicMock(return_value=original_req)

    # Verify the re-submitted request has embedding set and audio cleared

    assert resubmitted.input_embedding == fake_embedding_output
